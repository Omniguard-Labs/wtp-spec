# Security Policy

WTV is security-sensitive because it defines wallet transaction verification behavior.

## Reporting

Do not publish exploitable vulnerabilities in public issues.

Use GitHub private vulnerability reporting after the repository is public. If that is unavailable, contact a maintainer privately and share:

- affected spec section or SDK path;
- attack scenario;
- expected impact;
- suggested mitigation, if any.

## Public Discussion

Non-sensitive design concerns may be discussed in issues or discussions.

Examples:

- unclear verifier behavior;
- missing chain validation rule;
- interoperability failure;
- test vector mismatch.

## Supported Versions

Only the current draft branch is maintained until `WTV-v1` is frozen.
