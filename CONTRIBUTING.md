# Contributing

WTV is a draft standard. Contributions are welcome from wallet vendors, security companies, researchers, and independent implementers.

## How to Contribute

- Use issues for bugs, interoperability gaps, and security review notes.
- Use discussions for broad design questions.
- Use pull requests for concrete text, test vector, or reference implementation changes.
- Keep proposals small and reviewable.

## Spec Changes

A breaking spec change should include:

- problem statement;
- affected profiles;
- compatibility impact;
- verifier behavior;
- test vector impact.

## Pull Requests

Before opening a PR:

- run `npm test` when code or tests change;
- update relevant docs when behavior changes;
- avoid mixing unrelated spec and SDK changes.

## Intellectual Property

By contributing, you agree that your contribution is provided under the license of the file being modified.

If you represent a company, ensure you have permission to contribute.
